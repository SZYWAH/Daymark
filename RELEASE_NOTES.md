# Daymark 0.1.0-rc.2 发布说明

Daymark 是一个本地优先的个人知识库桌面应用，用来记录日常内容、整理资料、沉淀长期记忆，并在用户授权的范围内使用 AI 辅助整理和回顾。

## 版本状态

- 版本：0.1.0-rc.2
- 平台：Windows
- 安装包：NSIS `.exe`
- 状态：发布候选版，适合小范围试用和反馈

## 安装与安全提示

- 双击安装包后按安装向导完成安装。
- 当前版本未做代码签名，Windows 可能显示安全提醒；这是未签名应用的常见提示。
- 如需分发给他人，请发送安装包，不要发送 `target/debug` 或 `target/release` 下的开发产物。

## RC2 主要变化

- 新增自动工作回顾：用户明确开启后，Daymark 运行期间每 30 分钟增量整理当天 Codex 和 Claude Code 对话，也可手动立即更新。
- 自动工作回顾按天滚动保存为本地草稿，可在回顾档案中找回并保存为正式档案，不会自动进入长期记忆或核心备份。
- AI 回顾台与回顾档案支持应用内日历筛选，并增加“今日”快捷筛选。
- 顶部快速记录可直接打开并聚焦 Daymark 主窗口，同时保留未保存草稿。
- 桌面端手动 API Key 保存到系统凭据，并按供应商和 Base URL 隔离。
- 完成核心备份与覆盖恢复、会话只读预览、资料库 AI 操作可见性及内部模块稳定性改进。

## 数据与隐私

- Daymark 的内容保存在用户本机，不会主动上传到服务器。
- 桌面端手动填写的 API Key 保存到系统凭据存储。
- Web 模式仅作为本机开发降级路径，不建议用于共享设备。
- 整理、总结、回顾和提炼通常只在用户主动触发时调用 AI。
- 用户明确开启“自动工作回顾”后，Daymark 会先在本机对新增会话正文脱敏，再按固定周期发送给用户配置的 AI 服务；关闭后不再自动读取正文或调用 AI。

## 备份与恢复

- 设置页提供“数据备份”区块，可以手动导出核心备份。
- 核心备份包含资料、目录、日记、长期记忆文档、记忆卡片和内容链接。
- 核心备份不包含 API Key、AI 设置、AI 草稿、自动工作回顾草稿、历史总结报告、会话索引、主题或布局偏好。
- 恢复核心备份会覆盖当前核心内容，恢复前请确认备份文件来源可信。

## 反馈建议

反馈问题时建议说明：

- Windows 版本。
- Daymark 版本号。
- 发生问题的页面或操作步骤。
- 是否使用 AI 配置、核心备份或会话回顾功能。
- 如有报错，请提供脱敏后的错误信息截图或文字。
