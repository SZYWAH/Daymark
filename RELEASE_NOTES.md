# Daymark 0.1.0-rc.9 发布说明

Daymark 是一个本地优先的工作记忆工具，用来记录日常内容、整理资料、沉淀长期记忆，并在用户授权的范围内使用 AI 辅助整理和回顾。

## 版本状态

- 版本：0.1.0-rc.9
- 平台：Windows x64
- 安装包：NSIS `.exe`，内置 WebView2 离线运行时
- 状态：发布候选版，适合小范围试用和反馈

## 安装与安全提示

- 安装包内置 Microsoft Edge WebView2 Runtime，目标电脑安装时无需连接微软服务器下载该运行时。
- 可以直接覆盖安装此前 RC 版本，无需先卸载；建议升级前导出一次核心备份。
- 当前版本未做代码签名，Windows 可能显示安全提醒；这是未签名应用的常见提示。
- 请从本仓库的 GitHub Release 下载，并可使用发布页提供的 SHA256 校验值验证文件完整性。

## RC9 重点改进

- 修复设置页有未保存 AI 修改时无法前往其他页面的问题：现在可选择保存并前往、放弃修改并前往，或继续编辑。
- 设置页收纳 AI 连接为摘要与独立配置面板；外观、AI 连接、备份和自动化区块的留白与对齐同步整理。
- 自定义 AI 连接改由桌面端 Rust 网络层请求，减少第三方服务受 WebView CORS 限制导致的失败。
- OpenAI 兼容连接支持 Chat Completions 与 Responses；完整端点地址可原样使用，并提供模型列表获取与推理强度设置。
- 新增 Anthropic Messages 支持，覆盖官方服务与兼容网关的文本、流式响应和图片消息格式。
- API Key 继续保存到系统凭据，并按连接预设和规范化 Base URL 隔离；同一 OpenAI 地址的两种协议共享 Key。
- 顶部快速记录改为“薄入口后点击展开”：悬停不会再直接弹出大编辑窗，减少遮挡与误触；可在设置中关闭顶部入口，快捷键和托盘入口仍可用。
- Windows 安装包改用离线 WebView2 安装方式，改善断网或受限网络环境的首次安装成功率。

## RC8 延续修复

- 修复自定义标题栏遮挡“安静写作”、日志全屏查看与今日阅读层顶部内容的问题。
- 应用内全屏编辑与阅读层现在限制在标题栏下方的工作区，普通窗口和最大化窗口均可自适应。

## 现有核心能力

- 本地日志、资料库、长期记忆、内容关联与全文搜索。
- 核心数据备份和覆盖恢复。
- Codex 与 Claude Code 会话只读扫描、正文预览、AI 回顾和回顾归档。
- 用户主动开启后的自动工作回顾，以及顶部快速记录入口。
- 显示模式、六套主题风格与强调色自定义。

## 数据与隐私

- Daymark 的内容保存在用户本机，不会主动上传到服务器。
- 桌面端手动填写的 API Key 保存到系统凭据存储；真实 Key 不会回显、写入核心备份或错误正文。
- 整理、总结、回顾和提炼只在用户主动触发时调用 AI。
- 用户明确开启“自动工作回顾”后，Daymark 会先在本机对新增会话正文脱敏，再按固定周期发送给用户配置的 AI 服务；关闭后不再自动读取正文或调用 AI。

## 备份范围

- 核心备份包含资料、目录、日记、长期记忆文档、记忆卡片和内容链接。
- 核心备份不包含 API Key、AI 设置、AI 草稿、自动工作回顾草稿、历史总结报告、会话索引、主题或布局偏好。
- Daymark 内置示例资料及仅供示例使用的空目录不进入核心备份。
- 恢复核心备份会覆盖当前核心内容，恢复前请确认备份文件来源可信。

## 反馈建议

反馈问题时请说明 Windows 版本、Daymark 版本号、发生问题的页面和操作步骤；如有报错，请仅提供脱敏后的错误信息。
