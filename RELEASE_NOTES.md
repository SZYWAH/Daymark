# Daymark 0.1.0-rc.11 发布说明

Daymark 是一个本地优先的工作记忆工具，用来记录日常内容、整理资料、沉淀长期记忆，并在用户授权的范围内使用 AI 辅助整理和回顾。

## 版本状态

- 版本：0.1.0-rc.11
- 平台：Windows x64
- 安装包：NSIS `.exe`，提供内置 WebView2 的离线版和需要联网下载 WebView2 的在线小包
- 状态：发布候选版，适合小范围试用和反馈

## 安装与安全提示

- 安装包内置 Microsoft Edge WebView2 Runtime，目标电脑安装时无需连接微软服务器下载该运行时。
- Release 同时提供 `-setup-online.exe` 在线小包；目标电脑已有 WebView2 或可访问微软服务时可优先选择该版本。
- 可以直接覆盖安装此前 RC 版本，无需先卸载；建议升级前导出一次核心备份。
- 当前版本未做代码签名，Windows 可能显示安全提醒；这是未签名应用的常见提示。
- 请从本仓库的 GitHub Release 下载，并可使用发布页提供的 SHA256 校验值验证文件完整性。

## RC11 回顾沉淀、Markdown 与知识链接

- AI 每日回顾可以手动保存为资料：支持单来源、综合及已归档自动工作回顾，保存前可确认标题、正文、标签和目录；同一正式回顾不会重复创建资料。
- 回顾资料增加来源同步与版本闭环：正式回顾变化后可对比更新当前资料、另存新版本、重新启用已有来源版本，或把历史版本恢复为新的当前版本；搜索和普通列表只展示版本链当前项。
- 资料正文支持安全 Markdown 阅读与编辑，包括 GFM 表格、任务列表、脚注、代码块、基础工具栏、桌面分栏预览和窄屏编辑/预览切换；危险协议、原始 HTML 和非 HTTPS 网络图片不会被执行或加载。
- 新增稳定的资料内部链接与反向引用：编辑时可搜索并插入 `[[item:...]]`，阅读时可跳转、查看引用上下文与失效目标；回顾版本链链接会自动指向当前版本。
- 优化启动与 Today 数据加载：各模块分片加载，单项失败不再拖住整个应用；超时后会进入明确的可重试状态，刷新失败时保留上一次完整数据。
- 完善弹窗焦点循环、Escape 层级、焦点恢复、导航语义、对比度和页面主区域结构，改善键盘操作与辅助技术体验。
- 正文链接同步改为增量差异更新，200 条批量导入使用原子事务，减少大型资料库中无变化关系的重复写入。
- 桌面凭据命名空间按应用标识隔离，QA 环境默认只允许连接明确配置的本地 Mock 地址，降低测试环境误用正式凭据或远程端点的风险。

## RC11 验证范围

- 自动化类型检查、前端与 Rust 测试、数据不变量、Mock AI 协议、浏览器回归及隔离 QA 安装升级生命周期均已执行。
- 本版本仍是预发布候选版；托盘、系统文件选择器、不同 DPI 下的人工视觉以及真实 AI 服务未作为本轮全量自动化通过项，建议继续在小范围试用后再评估稳定版。

## RC10 会话扫描与 AI 回顾体验

- 会话扫描升级为应用级任务：开始扫描后自动进入独立进度页，收起或切换模块不会中断任务，并可随时返回查看状态。
- 大文件扫描期间持续展示真实阶段和计数，包括候选收集、已检查会话、已检查字节、缓存命中、精确命中、排除数量与耗时，不使用不可靠的整体百分比。
- 完善有日期、无日期与取消场景的扫描事件；取消或失败时保留上一次完整结果，并在任务结束后恢复后台日期索引补全。
- AI 回顾页收敛为唯一主操作，根据当前状态明确显示扫描、选择会话、配置 AI、生成回顾或查看进度，移除重复入口与需要重复点击的隐式确认。
- 每次生成统一使用一次确认弹窗；无日期或范围过大时在同一弹窗中显示风险说明，筛选或选择变化后会要求重新确认。
- 一次回顾只允许选择单一会话来源；首次勾选后锁定来源，全部取消时解除，并为混合来源结果提供清晰的同来源全选规则。
- 扫描与生成任务互斥，前台任务页和全局浮动入口按当前任务状态统一显隐，避免两个任务页或历史中断操作同时干扰当前流程。

## RC9.1 稳定性修复

- AI 配置的模型输入框与已获取模型列表合并：仍可手动填写模型，点击输入框内的箭头可选择已获取模型。
- 修复 AI 配置弹层输入框连续输入、删除、粘贴时意外丢失焦点的问题。
- 测试连接、获取模型和保存设置使用固定状态位与稳定滚动条空间，减少弹层状态切换时的视觉跳动。
- 保存 AI 配置后自动关闭配置弹层，并在设置页摘要中确认保存结果；失败时保留草稿与错误信息。

## RC9 重点改进

- 修复设置页有未保存 AI 修改时无法前往其他页面的问题：现在可选择保存并前往、放弃修改并前往，或继续编辑。
- 设置页收纳 AI 连接为摘要与独立配置面板；外观、AI 连接、备份和自动化区块的留白与对齐同步整理。
- 自定义 AI 连接改由桌面端 Rust 网络层请求，减少第三方服务受 WebView CORS 限制导致的失败。
- OpenAI 兼容连接支持 Chat Completions 与 Responses；完整端点地址可原样使用，并提供模型列表获取与推理强度设置。
- 新增 Anthropic Messages 支持，覆盖官方服务与兼容网关的文本、流式响应和图片消息格式。
- API Key 继续保存到系统凭据，并按连接预设和规范化 Base URL 隔离；同一 OpenAI 地址的两种协议共享 Key。
- 顶部快速记录改为“薄入口后点击展开”：悬停不会再直接弹出大编辑窗，减少遮挡与误触；可在设置中关闭顶部入口，快捷键和托盘入口仍可用。
- Windows 安装包改用离线 WebView2 安装方式，改善断网或受限网络环境的首次安装成功率。

## RC8 延续修复

- 修复自定义标题栏遮挡“安静写作”、日志全屏查看与今日阅读层顶部内容的问题。
- 应用内全屏编辑与阅读层现在限制在标题栏下方的工作区，普通窗口和最大化窗口均可自适应。

## 现有核心能力

- 本地日志、资料库、长期记忆、内容关联与全文搜索。
- 核心数据备份和覆盖恢复。
- Codex 与 Claude Code 会话只读扫描、正文预览、AI 回顾和回顾归档。
- 用户主动开启后的自动工作回顾，以及顶部快速记录入口。
- 显示模式、六套主题风格与强调色自定义。

## 数据与隐私

- Daymark 的内容保存在用户本机，不会主动上传到服务器。
- 桌面端手动填写的 API Key 保存到系统凭据存储；真实 Key 不会回显、写入核心备份或错误正文。
- 整理、总结、回顾和提炼只在用户主动触发时调用 AI。
- 用户明确开启“自动工作回顾”后，Daymark 会先在本机对新增会话正文脱敏，再按固定周期发送给用户配置的 AI 服务；关闭后不再自动读取正文或调用 AI。

## 备份范围

- 核心备份包含资料、目录、日记、长期记忆文档、记忆卡片和内容链接。
- 核心备份不包含 API Key、AI 设置、AI 草稿、自动工作回顾草稿、历史总结报告、会话索引、主题或布局偏好。
- Daymark 内置示例资料及仅供示例使用的空目录不进入核心备份。
- 恢复核心备份会覆盖当前核心内容，恢复前请确认备份文件来源可信。

## 反馈建议

反馈问题时请说明 Windows 版本、Daymark 版本号、发生问题的页面和操作步骤；如有报错，请仅提供脱敏后的错误信息。
