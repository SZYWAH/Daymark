# Daymark RC2 验收清单

这份清单用于 RC2 前的手动回归验收。重点覆盖近期改动过、最容易退化的桌面能力。

## 自动检查

- `pnpm typecheck`
- `pnpm test`
- `pnpm test:rust`
- `pnpm build`
- `git diff --check`

## 快速记录

- 主窗口最小化或隐藏后，顶部热区能唤出快速记录。
- 输入未保存草稿后点击“打开 Daymark”，主窗口打开并聚焦，快速记录自动收起。
- 再次打开快速记录时，未保存草稿仍然保留。
- 保存中不能打断“留下”流程，保存成功后主窗口能收到刷新事件。

## 今日页

- 无内容时，四个胶囊下方只显示一行轻量空状态文字。
- 保存一条此刻记录后，空状态文字消失，“今天已写”数量更新。
- 展开 `AI 今日回顾` 后，只显示轻量自动工作回顾状态，不常驻展示大卡片正文。
- 点击自动工作回顾“查看”可打开阅读层，关闭后回到今日页。

## 记忆页

- `AI 回顾台` 能扫描当天 Codex / Claude Code 会话。
- 起始日期、结束日期日历选择后，修改筛选会触发重新扫描保护。
- 放大会话列表预览中，来源、日期、搜索三个筛选控件高度一致。
- 点击单条会话“只读预览”只在本地读取正文，不保存、不上传、不触发 AI。
- `回顾档案` 的起始日期、结束日期日历筛选能处理空范围、单边范围和完整范围。
- 自动工作回顾草稿能在回顾档案中找回，并可保存为正式档案。

## 资料库

- 桌面端能够通过系统文件选择器导入资料，不依赖 Vite 开发服务器的动态模块缓存。
- 资料详情中的“总结、标题、标签、待办”四个 AI 操作在未悬停时也能清晰识别。
- 文件来源复制、打开和定位操作正常，AI 整理不会改变原始附件。

## 设置页

- 桌面端手动 API Key 保存到系统凭据，IndexedDB 不保留真实 Key。
- 不同 provider / Base URL 的 Key 不互相覆盖。
- 核心备份能导出和覆盖恢复，备份内容不包含真实 API Key、AI 草稿或自动工作回顾草稿。
- 自动工作回顾“立即更新”正常；应用保持运行 30 分钟后至少完成一次定时更新。

## 打包前确认

- Windows 未签名提示已在发布说明中说明。
- 构建产物、`dist/`、`src-tauri/target/`、安装包不提交到 Git。
- 版本号、产品名、窗口标题保持 Daymark RC 信息一致。
